Home page — user with account · on a team

Interactive map of index.html for a signed-in competitor whose squad has (or will have) approved tournament registrations. Each step explains what the user does, how the front end reacts, and which backend endpoints power the data. Scope is the home page only (not a full product walkthrough).

Prerequisite: Valid JWT in localStorage (wz_session) from POST /api/auth/login. Participant highlights require GET /api/players/me/participations to return at least one row with status: approved (via team captaincy or linked player roster).

Page load & data pipeline

Order matters: participations load first so featured cards and activity can mark “your” tournaments.

initHome() in index.html (inline script): 
await loadHomeParticipations();
await Promise.all([
  loadFeaturedTournaments(),
  loadGameSpotlight(),
  loadTodayMatches(),
  loadTournamentActivity(),
  loadAnnouncements()
]);
Also on parse: injectNav('index.html') → applyNavAuthState() reads session and shows Team / Player / username / Logout.
